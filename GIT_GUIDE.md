# Git Quick Start Guide for Software Requirements Classification Project

## What is Git & GitHub?

- **Git** — Version control system that tracks changes to your files
- **GitHub** — Cloud platform to host and share Git repositories
- **.gitignore** — File that tells Git which files NOT to track

## Step-by-Step Git Commands

### 1️⃣ INITIAL SETUP (First time only)

#### Install Git
```bash
# On Windows (if not installed)
# Download from https://git-scm.com/download/win
```

#### Configure Git
```bash
# Set your identity globally
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Verify
git config --global user.name
git config --global user.email
```

---

### 2️⃣ INITIALIZE LOCAL REPOSITORY

Navigate to your project directory:
```bash
cd d:\IDS_PROJECT
```

Initialize Git repository:
```bash
git init
```

This creates a hidden `.git` folder that tracks all changes.

---

### 3️⃣ CHECK STATUS

Always check what files Git has detected:
```bash
git status
```

**Output explanation:**
```
Untracked files:
  (use "git add <file>..." to include in what will be committed)
    PROJECT/3_EDA.py
    PROJECT/4_Data_Cleaning.py
    ...
```

---

### 4️⃣ ADD FILES TO STAGING

Add ALL files that respect `.gitignore`:
```bash
# Add all tracked files at once
git add .

# Or add specific files
git add PROJECT/3_EDA.py README.md
```

Check what's staged:
```bash
git status
```

You should see:
```
Changes to be committed:
  new file:   PROJECT/3_EDA.py
  new file:   README.md
  ...
```

---

### 5️⃣ COMMIT CHANGES

Create a snapshot with a descriptive message:
```bash
git commit -m "Add software requirements classification NLP pipeline with dashboard"
```

**Good commit message format:**
```
git commit -m "Brief description of changes"

Examples:
✅ "Add Streamlit dashboard for requirement classification"
✅ "Complete EDA with 3 key insights"
✅ "Add confusion matrix and feature importance analysis"
❌ "update"
❌ "fixes"
```

---

### 6️⃣ CREATE GITHUB REPOSITORY

1. Go to [GitHub.com](https://github.com)
2. Sign up or log in
3. Click **New Repository** (top right)
4. Name: `software-requirements-classification` (or similar)
5. Description: "NLP-based software requirements classification using PROMISE dataset"
6. Choose **Public** (for class submission) or **Private** (for personal)
7. DO NOT initialize with README (you already have one)
8. Click **Create Repository**

---

### 7️⃣ CONNECT LOCAL TO GITHUB

Link your local repository to GitHub:
```bash
# Copy this command from GitHub (replace USERNAME and REPO_NAME)
git remote add origin https://github.com/YOUR_USERNAME/software-requirements-classification.git

# Verify connection
git remote -v
```

Expected output:
```
origin  https://github.com/YOUR_USERNAME/software-requirements-classification.git (fetch)
origin  https://github.com/YOUR_USERNAME/software-requirements-classification.git (push)
```

---

### 8️⃣ PUSH TO GITHUB

Upload your local repository to GitHub:
```bash
# Push to 'main' branch (default)
git branch -M main
git push -u origin main
```

**First time might ask for authentication:**
- GitHub username or personal access token
- Follow the on-screen instructions

---

### 9️⃣ VERIFY ON GITHUB

1. Go to your GitHub repository URL
2. You should see all your `.py` files, README, dataset, and figures
3. `.gitignore` ensures large `.pkl`, `.npz`, `.npy` files are NOT uploaded

## Common Workflow After Initial Push

### Make changes locally, then push:

```bash
# 1. Make changes to files
# (Edit your Python code, update README, etc.)

# 2. Check what changed
git status

# 3. Stage changes
git add .

# 4. Commit with message
git commit -m "Fix bug in feature engineering step"

# 5. Push to GitHub
git push
```

## Useful Git Commands

| Command | Purpose |
|---------|---------|
| `git status` | See untracked/modified files |
| `git add .` | Stage all changes |
| `git add filename` | Stage specific file |
| `git commit -m "msg"` | Create checkpoint |
| `git push` | Upload to GitHub |
| `git pull` | Download latest from GitHub |
| `git log` | View commit history |
| `git diff` | See line-by-line changes |
| `git reset HEAD file` | Unstage file |
| `git checkout -- file` | Discard local changes |

## Handling Generated Files

### Scenario: Model trained locally, .pkl files created

```bash
# These files appear but should NOT be committed
ls -la *.pkl
# Output: requirement_classifier.pkl, tfidf_vectorizer.pkl

# Check git status
git status
# These won't show because .gitignore blocks them ✅

# If they were accidentally tracked, remove them:
git rm --cached *.pkl
git commit -m "Stop tracking large model files"
git push
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Not a git repository" | Run `git init` in project directory |
| "Permission denied" | Use GitHub personal access token instead of password |
| "Large files rejected" | Ensure .gitignore is working (git status should hide .pkl files) |
| "Merge conflict" | Don't worry yet — only relevant if multiple people edit same file |
| "Want to undo commit" | `git reset HEAD~1` (keeps changes) or `git revert HEAD` (keeps history) |
